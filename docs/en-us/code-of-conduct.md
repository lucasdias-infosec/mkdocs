# Code of Conduct

## 1. Scope and Purpose

This Code of Conduct defines the behavioral and professional standards expected from individuals participating in this project.

Its objective is to ensure a technically focused, respectful, and secure collaboration environment consistent with professional practices in Information Security.

This Code applies to all contributors, reviewers, collaborators, and participants.

---

## 2. Professional Conduct

Participants are expected to:

- Maintain professional and respectful communication.
- Provide technically substantiated and constructive feedback.
- Keep discussions focused on project objectives and technical merit.
- Respect differing technical opinions and levels of expertise.
- Act in accordance with generally accepted ethical standards in Information Security.

---

## 3. Prohibited Conduct

The following behaviors are not permitted:

- Personal attacks or hostile interactions.
- Harassment or intimidation.
- Disruptive conduct that impairs technical discussion.
- Submission of malicious, intentionally vulnerable, or deceptive code.
- Inclusion of confidential, proprietary, or real production data without authorization.

---

## 4. Security and Data Responsibility

Given the technical nature of this project, contributors must:

- Avoid committing credentials, tokens, or sensitive information.
- Avoid publishing data derived from real production systems.
- Follow responsible disclosure practices when reporting vulnerabilities.
- Clearly document laboratory or proof-of-concept configurations.

Maintaining security integrity is a shared responsibility.

---

## 5. Contribution Standards

Contributions should:

- Follow the documented contribution guidelines.
- Complete the provided Pull Request and Issue templates when applicable.
- Provide clear descriptions, reproducible steps, and sufficient technical context.
- Maintain consistency in terminology and documentation quality.
- Contributors must also follow the documented Contribution Guide when submitting changes.

Submissions lacking essential technical information may be returned for clarification or revision.

Contributions that do not meet established technical or security standards may be declined.

---

## 6. Reporting and Enforcement

Concerns regarding violations of this Code may be reported to the project maintainer.

The maintainer reserves the right to:

- Request modifications to contributions.
- Remove inappropriate content.
- Restrict participation when necessary to preserve project integrity.

Enforcement decisions will prioritize technical quality, security standards, and professional conduct.

---

## 7. Acceptance

Participation in this project implies acceptance of this Code of Conduct.
